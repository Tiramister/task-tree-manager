## Why

現在の JSON インポート要件はフロントエンド側の置換を中心に定義されており、ログイン利用時にバックエンド側タスクデータをどのように扱うかが明確でない。インポート時に警告を出した上でバックエンドデータも全件上書きする挙動を明確化し、データ不整合と予期しない復元を防ぐ必要がある。

## What Changes

- JSON インポート実行前に、既存データが失われることを明示する警告確認を必須化する。
- ユーザーが警告を承認した場合、インポートデータでフロントエンドのタスク状態を置換する。
- ログイン済みユーザーでは、インポート後にバックエンドの当該ユーザータスクを全件インポートデータで上書きする。
- 上書き処理中に失敗した場合の整合性要件（部分反映を避ける、または失敗時の扱いを明確化）を定義する。

## Capabilities

### New Capabilities
- なし

### Modified Capabilities
- `data-import`: インポート時の警告確認と、承認後にインポートデータで置換する要件をバックエンド上書きまで含めて明確化する。
- `task-sync`: JSON インポートを起点にバックエンドタスク全件をインポートデータで上書き同期する要件を追加する。

## Impact

- フロントエンド: インポートフローの確認 UI、インポート後同期処理、エラーハンドリング。
- バックエンド API: タスク全件上書きを実現するための API 呼び出し手順または専用手段の定義。
- データ整合性: ローカル状態・バックエンド状態・次回ログイン時同期挙動の一貫性要件。
- テスト: インポート承認/キャンセル、上書き成功/失敗、同期後の取得結果に関する統合テスト。
