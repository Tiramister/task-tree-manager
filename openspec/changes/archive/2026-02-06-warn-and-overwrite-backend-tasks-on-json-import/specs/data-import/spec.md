## MODIFIED Requirements

### Requirement: インポートは既存データを置換する

インポートを実行すると、既存のすべてのタスクデータが削除され、インポートデータで置き換えられなければならない（SHALL）。未ログイン時はローカルデータのみを置換し、ログイン済み時はバックエンドの当該ユーザータスクもインポートデータで全件上書きしなければならない（SHALL）。

#### Scenario: 未ログイン状態でインポートする
- **WHEN** 未ログイン状態で既存タスクがある中、インポートを実行する
- **THEN** ローカルの既存タスクがすべて削除され、インポートしたタスクのみが存在する

#### Scenario: ログイン済み状態でインポートする
- **WHEN** ログイン済み状態で既存タスクがある中、インポートを実行する
- **THEN** ローカルの既存タスクがすべて削除され、バックエンドの当該ユーザータスクも全件がインポートデータで上書きされる

#### Scenario: バックエンド上書きに失敗する
- **WHEN** ログイン済み状態でインポート後のバックエンド上書き処理が失敗する
- **THEN** インポート結果のローカルデータは維持され、同期失敗のエラーが表示される

### Requirement: インポート前に確認ダイアログを表示する

既存データが失われることをユーザーに警告するため、インポート実行前に確認ダイアログを表示しなければならない（SHALL）。確認文言には、ログイン済み時にバックエンドデータも上書きされることを含めなければならない（SHALL）。

#### Scenario: ログイン済みユーザーが確認ダイアログで OK を選択する
- **WHEN** ログイン済みユーザーがインポート確認ダイアログで OK を選択する
- **THEN** インポートが実行され、ローカルおよびバックエンドのデータ上書き処理が開始される

#### Scenario: 確認ダイアログでキャンセルを選択する
- **WHEN** ユーザーがインポート確認ダイアログでキャンセルを選択する
- **THEN** インポートは実行されず、既存データが維持される
