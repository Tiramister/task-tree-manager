## Context

タスクツリー管理アプリの基本機能（Phase 1〜6）は完成済み。初回起動時にデータが空であるため、ユーザーが使い方を理解しにくい状態。仕上げとして、サンプルデータの投入と UI/UX の微調整を行う。

## Goals / Non-Goals

**Goals:**
- 初回起動時にサンプルタスクを表示し、アプリの使い方を直感的に理解できるようにする
- 空状態時に適切なガイドメッセージを表示する
- 全体的な UI の完成度を上げる

**Non-Goals:**
- サンプルデータのカスタマイズ機能
- オンボーディングツアーやチュートリアル
- パフォーマンス最適化

## Decisions

### サンプルデータの投入方法

**決定**: Zustand store の初期値として、localStorage にデータがない場合にサンプルデータを設定する。

**理由**: 既存の persist ミドルウェアの仕組みを活用でき、localStorage にデータがあれば自動的にそちらが優先される。追加ロジックが最小限で済む。

**代替案**:
- 初回起動フラグを別途管理 → 不要な複雑さ
- ボタンで明示的に投入 → 初回体験が空のまま

### 空状態の表示

**決定**: タスクが0件の場合、ツリービュー部分にメッセージとタスク作成ボタンを表示する。

**理由**: ユーザーがサンプルデータを削除した後も、次のアクションが明確になる。

## Risks / Trade-offs

- サンプルデータの削除後は元に戻せない → 空状態メッセージで補完
- サンプルデータが多すぎると邪魔 → 最小限の階層構造（2〜3階層、5〜8タスク程度）に抑える
