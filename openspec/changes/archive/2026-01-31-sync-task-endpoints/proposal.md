## Why

フロントエンドのタスクデータは現在 localStorage に保存されており、サーバーサイドとの同期ができていない。バックエンドにはすでにタスクテーブル（`tasks`）と認証機構が存在するため、タスクの CRUD エンドポイントを実装し、フロントエンドのデータをユーザーごとにサーバーサイドで永続化できるようにする。

## What Changes

- タスクの一覧取得エンドポイント（`GET /tasks`）を追加
- タスクの作成エンドポイント（`POST /tasks`）を追加
- タスクの更新エンドポイント（`PATCH /tasks/{id}`）を追加
- タスクの削除エンドポイント（`DELETE /tasks/{id}`）を追加
- タスクの並び替えエンドポイント（`PATCH /tasks/{id}/reorder`）を追加
- すべてのエンドポイントは認証済みセッションを必要とし、ログインユーザーのタスクのみ操作可能

## Capabilities

### New Capabilities

- `task-api`: タスクの CRUD 操作および並び替えを行う REST API エンドポイント群。認証済みユーザーのタスクに対する一覧取得・作成・更新・削除・並び替えをカバーする。

### Modified Capabilities

（なし）

## Impact

- **バックエンド（Go）**: `main.go` にルーティング追加、タスク操作用のハンドラーファイルを新規作成
- **API**: 5 つの新規エンドポイントが追加される
- **データベース**: 既存の `tasks` テーブルをそのまま利用（スキーマ変更なし）
- **認証**: 既存の `authMiddleware` で保護される（変更なし）
