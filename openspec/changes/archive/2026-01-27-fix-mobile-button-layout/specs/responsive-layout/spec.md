## MODIFIED Requirements

### Requirement: モバイル向けレイアウト調整
アプリケーションは画面幅に応じてレイアウトを動的に調整しなければならない（SHALL）。モバイル（640px未満）ではパディングを縮小し、コンテナの最大幅制限を解除しなければならない。ツリービュー上部のボタン群は横幅が不足する場合に折り返さなければならない（SHALL）。

#### Scenario: モバイル画面でのレイアウト
- **WHEN** 画面幅が640px未満の場合
- **THEN** コンテナのパディングが縮小され（px-2）、最大幅制限が解除される

#### Scenario: デスクトップ画面でのレイアウト
- **WHEN** 画面幅が640px以上の場合
- **THEN** 従来通りのパディング（px-4）と最大幅制限（max-w-3xl）が適用される

#### Scenario: モバイル画面でのボタン群の折り返し
- **WHEN** 画面幅が狭くボタン群が1行に収まらない場合
- **THEN** ボタン群が自動的に折り返され、すべてのボタンが画面内に表示される
