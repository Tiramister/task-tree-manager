## ADDED Requirements

### Requirement: JSON ファイルからタスクデータをインポートできる

ユーザーが JSON ファイルを選択すると、ファイルの内容がタスクデータとして読み込まれなければならない（SHALL）。

#### Scenario: 有効な JSON ファイルをインポートする
- **WHEN** 有効なタスクデータを含む JSON ファイルを選択する
- **THEN** ファイルのタスクデータがストアに読み込まれる

### Requirement: インポートは既存データを置換する

インポートを実行すると、既存のすべてのタスクデータが削除され、インポートデータで置き換えられなければならない（SHALL）。

#### Scenario: 既存データがある状態でインポートする
- **WHEN** 既存タスクが存在する状態でインポートを実行する
- **THEN** 既存タスクがすべて削除され、インポートしたタスクのみが存在する

### Requirement: インポート前に確認ダイアログを表示する

既存データが失われることをユーザーに警告するため、インポート実行前に確認ダイアログを表示しなければならない（SHALL）。

#### Scenario: 確認ダイアログで OK を選択する
- **WHEN** インポート確認ダイアログで OK を選択する
- **THEN** インポートが実行される

#### Scenario: 確認ダイアログでキャンセルを選択する
- **WHEN** インポート確認ダイアログでキャンセルを選択する
- **THEN** インポートは実行されず、既存データが維持される

### Requirement: 不正なファイルはエラーとして処理する

JSON として解析できないファイル、またはタスクの必須フィールド（id, title, status, createdAt）が欠けているデータはエラーとして処理しなければならない（SHALL）。

#### Scenario: JSON として解析できないファイルをインポートする
- **WHEN** JSON として解析できないファイルを選択する
- **THEN** エラーメッセージが表示され、既存データは変更されない

#### Scenario: 必須フィールドが欠けたデータをインポートする
- **WHEN** タスクの必須フィールドが欠けた JSON ファイルを選択する
- **THEN** エラーメッセージが表示され、既存データは変更されない
