## Context

タスク一覧（`TaskTreeItem`）では現在、期限の緊急度を視覚的に区別する仕組みがない。`Task` 型には `dueDate`（ISO 8601）と `status`（4種類）が既に存在するため、表示レイヤーの変更のみで実現可能。

## Goals / Non-Goals

**Goals:**

- 期限が近いタスクの背景色を緊急度に応じて段階的に表示する
- 完了以外のステータスのタスクのみを対象とする

**Non-Goals:**

- 通知やアラート機能の追加
- 期限の閾値をユーザーがカスタマイズできるようにすること
- 完了タスク履歴ビューへの適用

## Decisions

### 背景色のロジックをユーティリティ関数として切り出す

**決定**: 期限までの残り日数から背景色の Tailwind クラスを返すユーティリティ関数を作成する。

**理由**: `TaskTreeItem` コンポーネント内にロジックを直接書くと可読性が下がるため。また、将来的に他の箇所でも再利用しやすい。

**関数シグネチャ**:
```ts
function getDeadlineHighlightClass(dueDate: string | undefined, status: TaskStatus): string
```

**戻り値**:
- 当日以前（残り0日以下）: `"bg-red-50"`
- 2日以内（残り1〜2日）: `"bg-orange-50"`
- 6日以内（残り3〜6日）: `"bg-yellow-50"`
- 上記以外・期限なし・完了: `""` （空文字列）

### 日付比較のロジック

**決定**: `dueDate` と現在日付を日単位（時刻を無視）で比較する。

**理由**: `dueDate` は日付のみの入力（`<input type="date">`）であり、時刻を含めた比較は不適切。

**実装方針**: 両方の日付を `YYYY-MM-DD` 文字列として比較するか、日付の先頭をパースして差分日数を計算する。

## Risks / Trade-offs

- **[薄い背景色のコントラスト]** → `bg-*-50` はTailwind の最も薄い色であり、十分に控えめ。視認性に問題があれば `bg-*-100` に調整可能。
- **[ダークモード非対応]** → 現時点ではダークモード未実装のため考慮しない。将来対応時に `dark:` プレフィックスを追加すればよい。
