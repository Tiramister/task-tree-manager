## Why

現在のアプリケーションにはユーザー認証機能がなく、誰でもすべてのデータにアクセスできる状態である。ID/パスワードによるユーザー認証を導入し、認証済みユーザーのみがアプリケーションを利用できるようにする。

## What Changes

- PostgreSQL に `users` テーブルを追加し、ユーザー情報（ID・パスワードハッシュ）を保存する
- HTTP サーバーにログインエンドポイント（`POST /login`）を追加する
- HTTP サーバーにログアウトエンドポイント（`POST /logout`）を追加する
- セッション管理により、認証状態を維持する
- ユーザー追加のエンドポイントは作成しない（ユーザーは DB に直接登録する運用とする）

## Capabilities

### New Capabilities

- `user-auth`: ユーザー認証の仕組み全体をカバーする。users テーブルの定義、パスワードのハッシュ化、ログイン・ログアウトのエンドポイント、セッション管理を含む。

### Modified Capabilities

- `http-server`: 認証エンドポイント（`/login`, `/logout`）の追加、および認証ミドルウェアによる既存エンドポイントの保護。
- `database-migration`: users テーブル作成のマイグレーション追加。

## Impact

- **バックエンド**: `backend/main.go` にエンドポイントとミドルウェアを追加。`backend/migrations/` に新しいマイグレーションファイルを追加。
- **依存関係**: パスワードハッシュ化ライブラリ（`golang.org/x/crypto/bcrypt` など）の追加が必要になる可能性がある。
- **データベース**: `users` テーブルの新規作成。セッション管理方式によっては `sessions` テーブルも必要。
- **API**: `/login`, `/logout` エンドポイントの追加。既存エンドポイントは認証必須となる可能性がある。
