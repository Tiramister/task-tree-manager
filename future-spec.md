# アプリケーション概要
階層構造（木構造）でタスクを管理するWebアプリケーション

## 主要機能
### 1. 全体ビュー画面
タスクの階層構造を視覚的に表示
表示形式：フォルダーのように、親タスクの中に子タスクが入ることで表現する
折り畳み機能
* 個別のタスクの子を折り畳み/展開
* 全体の一括折り畳み/展開

### 2. タスク詳細画面
全体ビューでタスクをクリックしたとき、ドロワー形式で右から表示する

タスクデータ項目
* タスク名
* 詳細説明
* 期限
* ステータス（未着手/作業中/待ち/完了）
* 完了日
* 作業記録（メモ）

作業記録は当面はプレーンテキスト（将来的にMarkdown対応も検討）

## 技術要件
Webアプリケーション（ブラウザで動作）
レスポンシブデザイン（スマホブラウザでも快適に動作）
シングルユーザー想定

## 技術選定

フロントエンド

| 項目 | 選定 | 理由 |
| :--- | :--- | :--- |
| フレームワーク | React + TypeScript | コンポーネント設計が木構造と相性が良い、型安全性 |
| ビルドツール | Vite | 高速な開発体験、現在の主流 |
| UIライブラリ | Shadcn/ui + Tailwind CSS | カスタマイズ性が高く、ドロワー・ツリー系コンポーネントが揃っている |
| 状態管理 | Zustand | シンプルで軽量、シングルユーザーには十分 |

データ永続化

| 項目 | 選定 | 理由 |
| :--- | :--- | :--- |
| ストレージ | localStorage | シングルユーザー想定で十分、バックエンド不要で手軽 |

※将来的にデータ量が増える場合は IndexedDB への移行も可能

理由補足

- バックエンドなしでブラウザ完結型にすることで、初期構築がシンプルになる
- Shadcn/ui は Radix UI ベースで、Sheet（ドロワー）コンポーネントが標準で提供されている
- 木構造の表示は自前実装でも十分対応可能（再帰コンポーネント）
