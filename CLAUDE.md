# CLAUDE.md

## 開発方針
- ライブラリやフレームワークの設定ファイルは、可能な限りそのライブラリ・フレームワークが提供する初期化方法で生成してください。
    - 自分で設定ファイルを実装するのは最後の手段だと思ってください。
- 設定はなるべくライブラリ・フレームワークが初期化したものをそのまま使い、編集は最小限にしてください。
    - 必須でない設定でも、世間にベストプラクティスとして広まっている・公式が推奨している場合は設定してください。

## 開発フロー
- 実装後は必ず `npm run check` と `npm run build` を実行して、エラーがないことを確認してください。

## ツール
- ランタイム (Node.js など) のバージョン管理には、既にインストールされている mise-en-place を使ってください。
- grep の代わりに rg を使ってください。
